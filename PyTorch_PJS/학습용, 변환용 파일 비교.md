**Torch_Train.py는 학습용 파일, convert_pth_to_pt.py는 학습 끝난 .pth 모델을 .pt로 변환하는 전용 파일**


# 핵심 차이
| 구분 | Torch_Train.py | convert_pth_to_pt.py |
| -- | -- | -- |
| 주 목적 | Faster R-CNN 모델 학습 | .pth -> .pt 변환 |
| 데이터 사용 | 이미지/XML 데이터셋 사용 | 데이터셋 사용 안 함 |
| 학습 여부 | 학습함 | 학슴 안 함 |
| 증강 여부 | train 데이터에 증강 적용 | 증강 없음 |
| 평가 여부 | valid 데이터로 mAP 평가 | 평가 없음 |
| 저장 결과| .pth 마지막에 .pt 저장시도 | .pt 저장 |
| 실행 방식 | 학습 전체 파이프라인 | 변화만 수행하는 CLI도구 |
| 모델 입력 | train/valid 이미지 | 더미 이미지 1장 | 
| TorchScript 방식 | torch.jit.script(model) | torch.jit.trace(wrapper,dummy_inpit) |


# Torch_Train.py의 용도 
- 이 파일은 처음부터 모델을 학습시키는 코드 

### 흐름
- 1. 이미지 / XML 데이터 준비 
    - ↓
- 2. train/valid 폴더 분리 
    - ↓
- 3. train 데이터 증강
    - ↓
- 4. Faster R-CNN ResNet50 FPN 생성
    - ↓
- 5. 학습
    - ↓
- 6. valid 데이터로 mAP 평가 
    - ↓
- 7. best.pth 저장
    - ↓
- 8. last.pth 저장
    - ↓
- 9. 마지막에 .pt 저장 시도 

**즉, Torch_Train.py는 실제로 학습 돌릴 때 쓰는 메인 파일**

- 이 안에는 이런 기능들이 있음
    - VOC XML 파싱
    - Custom Dataset
    - train/valid 자동 분리
    - train 전용 증강 이미지 저장
    - DataLoader 생성
    - Faster R-CNN 학습
    - TensorBoard loss 기록
    - Profiler 기록
    - valid mAP 평가
    - best / last checkpoint 저장

- 그리고 마지막 부분에서 .pt 저장 시도


# convert_pth_to_pt.py 용도
- 이 파일은 이미 학습 끝난 .pt 파일을 .pt로 변환하는 전용 파일

### 흐름
- 1. .pth 파일 입력
    - ↓
- 2. Faster R-CNN 구족 생성
    - ↓
- 3. .pth 가중치 로드
    - ↓
- 4. 추론용 wrapper 적용
    - ↓
- 5. 더미 이미지 생성
    - ↓
- 6. torch.jit.trace() 실행 
    - ↓
- 7. .pt 저장

**즉, 학습은 전혀 하지 않고, 이미 만들어진 모델만 변환**

- 예를 들면 Torch_Train.py에서 만든: 
    - checkpoints/fasterrcnn_best.pth
    - 또는
    - checkpoints/fasterrcnn_custom_last.pth
    - 이런 파일을 받아서 :
        - checkpoints/fasterrcnn_best.pt
        - 형태로 바꾸는 역할


# 가장 중요한 차이 : .pt 저장 방식

## 1. Torch_Train.py
- Torch_Train.py는 Faster R-CNN 모델 자체를 바로 script 하려함

- 문제는 Faster R-CNN은 내부 구조가 복잡하고 출력도 dictionary 형태라서 torch.jit.script()에서 오류가 날 가능성이 있음
    - 그래서 코드에도 try/except가 있음

- 즉, .pt 저장이 실패할 수 있다는 전제가 들어가 있음


## 2. convert_pth_to_pt.py
- 이 파일은 모델을 바로 변환하지 않고 FasterRCNNWrapper로 감 쌈

- Wrapper는 Faster R-CNN의 복잡한 출력 중 필요한 것만 꺼냄
    - 그래서 최종 .pt 모델 출력은 단순해짐
        - 이 방식이 추론용으로는 더 깔끔


## 3. 둘의 관계 
- 순서상 연결되는 파일

- 전체 흐름:
    - 1. Torch_Train.py 실행
        - → 모델 학습 
        - → fasterrcnn_best.pth 또는 fasterrcnn_custom_last.pth 생성

    - 2. convert_pth_to_pt.py 실행
        - → 위에서 만든 .pth를 읽음
        - → 추론용 .pt 생성

- 즉, Torch_Train.py가 모델을 만드는 파일, convert_pth_to.py 가 만들어진 모델을 배포/추론용으로 바꾸는 파일


### 4. 쉽게 비유하면
- Torch_Train.py
    - 모델을 학습시키는 파일

- convert_pth_to_pt.py
    - 학습이 끝난 모델을 실제 추론 프로그램에서 쓰기 좋은 형태로 포장하는 파일


### 5.결론
- Torch_Train.py는 학습 + 검증 + checkpoint 저장까지 하는 전체 학습 파이프라인이고,
- convert_pth_to_pt.py는 학습된 .pth 모델을 .pt TorchScript 모델로 변환하는 후처리 도구
